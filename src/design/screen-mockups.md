## Design Implementation Notes

### Senior App Priorities:
1. **Every screen has ONE primary action** - never more than 2-3 buttons
2. **Back button always 80x80pt** in top left
3. **All text minimum 24pt** - no exceptions
4. **High contrast everywhere** - test with contrast checker
5. **Touch targets minimum 80pt** with 16pt spacing between
6. **Emergency button** always accessible (home screen or global)
7. **Simple confirmation** for all important actions
8. **Large success feedback** with green checkmarks
9. **No hidden menus** - everything visible upfront
10. **Voice feedback** for all critical confirmations

### Family App Priorities:
1. **Critical info at top** - SOS alerts, missed meds
2. **Bottom tabs for main navigation** - 5 tabs max
3. **Cards for all content** - consistent spacing (16pt padding)
4. **Real-time indicators** - online status, badges
5. **Quick actions everywhere** - swipe, long-press
6. **Search + filters** for long lists
7. **Floating action button** for primary actions
8. **Pull to refresh** on all lists/feeds
9. **Skeleton screens** while loading
10. **Haptic feedback** for important actions

### Responsive Considerations:
- Small phones (<375px): Reduce spacing by 25%
- Large phones (>414px): Increase spacing by 15%
- Tablet layouts: Use two-column layouts for Family app
- Landscape: Adjust home button grid to 4x1 for Senior app

### Accessibility Features:
- Screen reader labels on all interactive elements
- VoiceOver/TalkBack announcements for state changes
- Dynamic text sizing support (with limits on Senior app)
- High contrast mode support
- Reduced motion support
- Color-blind friendly color choices
